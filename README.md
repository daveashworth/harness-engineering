# Harness Engineering Skill

This repository contains the `harness-engineering` agent skill: a concise guide for setting up and maintaining agent-first software repositories.

The skill is based on OpenAI's harness engineering approach: humans steer, agents execute, and the repository becomes the system of record for maps, constraints, feedback loops, runtime evidence, and recurring cleanup.

Use it when you want to:

- bootstrap a new repo for coding-agent work
- add or improve `AGENTS.md`, `ARCHITECTURE.md`, and repo-local docs
- enforce architectural boundaries with lint rules or structural tests
- make an app runnable, observable, and verifiable by agents
- migrate an existing project toward safer agent autonomy

The canonical skill content lives in [`SKILL.md`](./SKILL.md).
